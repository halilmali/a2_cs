Section 1.1

Why Do We Need Protocols?

📖 Definition

A protocol is a set of rules that govern how data is communicated between devices on a network.

🧠 Analogy — The Postal Service

Imagine sending a letter without agreed rules:

  • What language is the address written in?
  • Where does the stamp go?
  • Is the return address required?
  • How does the post office know where to route it?

Protocols are the agreed rules that all participants follow — just like postal regulations ensure your letter arrives correctly.

⚠️ Problems WITHOUT a Protocol

ProblemConsequence
Data format mismatchSender sends binary, receiver expects text — data is gibberish
No error checkingCorrupted data is accepted as valid — silent data corruption
No agreed speed/timingSender blasts data at 1 Gbps, receiver reads at 1 Kbps — data lost
Can't detect start/endReceiver can't tell where one message ends and the next begins
No securityNo encryption, authentication, or access control — anyone can read/intercept
Protocol — A set of rules governing data communication between devices
Exam Tip

Common question: "Explain why a protocol is necessary for communication between two computers." Structure your answer: (1) Define protocol — agreed set of rules. (2) Mention format — ensures both devices interpret data the same way. (3) Mention timing/flow control — prevents data loss from speed mismatch. (4) Error checking — detects/corrects corrupted data.

Section 1.2

The Concept of a Protocol Stack

❓ Why Layers?

Rather than one giant, complex protocol, network communication is divided into layers. Each layer:

  • Performs one specific job (separation of concerns)
  • Only communicates with the layer directly above or directly below it
  • Can be developed, updated, or replaced independently

✅ Benefits of Layering

Modularity
Each layer handles one task
Easy troubleshooting
Isolate problems to one layer
Interchangeability
Swap out technologies (e.g., Wi-Fi instead of Ethernet at Link layer)
Standardisation
Different manufacturers can build compatible products
Each layer communicates only with its immediate neighbours — each performs one specific job
Exam Tip

If a question asks "Why are protocols organised into layers?", mention: modularity (each layer has one job), independent development (layers can be updated separately), easier troubleshooting (isolate faults to a specific layer), and interoperability (different vendors can build compatible products).

Protocol Stack / Layering — Organising protocols into independent layers, each with a specific function

Introducing the TCP/IP Protocol Suite

🌐 What is TCP/IP?

The TCP/IP protocol suite is the set of communication protocols used to connect devices across the Internet. It consists of 4 layers.

  • TCP = Transmission Control Protocol
  • IP = Internet Protocol
  • Developed in the 1970s; the foundation of the modern internet

🧩 OSI Model — Extension Note

There is also the OSI (Open Systems Interconnection) model with 7 layers. TCP/IP combines some OSI layers:

TCP/IP Stack — Layer 4 (Top)

Layer 1: Application Layer

📌 Function

The Application Layer is the topmost layer in the TCP/IP stack. It:

  • Allows applications (browsers, email clients, FTP clients) to access network services
  • Formats data so it is understandable by the user application
  • Provides the interface between the user's software and the network
  • Contains many high-level protocols (HTTP, FTP, SMTP, etc.)

🧪 Example: Web Browser Requesting a Webpage

Chrome HTTP Request Web Server

The browser (application) uses HTTP at the Application Layer to format and send the request for a webpage.

📋 Protocols at the Application Layer

ProtocolPurpose
HTTPTransferring web pages (HyperText Transfer Protocol)
HTTPSSecure version of HTTP (with SSL/TLS encryption)
FTPFile transfer between client and server (File Transfer Protocol)
SMTPSending email messages (Simple Mail Transfer Protocol)
POP3Receiving/retrieving email from a server (Post Office Protocol v3)
IMAPReceiving email with multi-device sync (Internet Message Access Protocol)
BitTorrentPeer-to-peer file sharing
Remember: The Application Layer does NOT refer to the application itself (like Chrome). It refers to the protocols that applications use to communicate over the network.
Application Layer — Provides network services to user applications; formats data for the user

TCP/IP Stack — Layer 3

Layer 2: Transport Layer

📌 Function

The Transport Layer is responsible for end-to-end delivery of data between two devices. It:

  • Breaks data from the Application Layer into smaller segments
  • Adds port numbers to identify which application the data belongs to
  • Provides flow control (regulates data speed)
  • Performs error checking (detects corrupted segments)

🔑 Port Numbers

Each application is assigned a port number so the Transport Layer knows which application to deliver data to:

  • HTTP → Port 80
  • HTTPS → Port 443
  • FTP → Port 21
  • SMTP → Port 25

TCP/IP Stack — Layer 2

Layer 3: Internet Layer

📌 Function

The Internet Layer (also called the Network Layer) is responsible for:

  • Logical addressing — adding source and destination IP addresses
  • Routing — determining the best path for data to travel across networks
  • Packaging data from the Transport Layer into packets (also called datagrams)

📮 IP Protocol

The main protocol at this layer is IP (Internet Protocol).

  • IPv4: 32-bit address (e.g., 192.168.1.1) — ~4.3 billion addresses
  • IPv6: 128-bit address (e.g., 2001:db8::1) — huge address space
  • IP is connectionless — each packet is routed independently
  • IP is unreliable — it does NOT guarantee delivery (that's TCP's job!)
The Internet Layer adds source/destination IP addresses and routes packets across networks
Key insight: IP addresses identify a device on a network. Port numbers (Transport Layer) identify the application on that device. Both are needed for data to reach the right place.
Internet Layer — Handles logical addressing (IP addresses) and routing of packets

TCP/IP Stack — Layer 1 (Bottom)

Layer 4: Link Layer

📌 Function

The Link Layer (also called the Network Interface or Data Link layer) is the bottom layer of the TCP/IP stack. It:

  • Handles physical addressing using MAC addresses
  • Controls transmission of data over the physical medium (cable, fibre, Wi-Fi)
  • Converts packets from the Internet Layer into frames for transmission
  • Deals with hardware protocols (Ethernet, Wi-Fi / 802.11)

🔗 MAC Address vs IP Address

Link Layer wraps the packet in a frame with source/destination MAC addresses for local delivery

🌐 Hardware Protocols at this Layer

  • Ethernet — wired networking standard for LANs
  • Wi-Fi (IEEE 802.11) — wireless networking standard
  • ARP (Address Resolution Protocol) — maps IP addresses to MAC addresses
Link Layer — Handles physical addressing (MAC) and transmission over physical media

Process Overview

Sending a Message Across the Internet

At Host A, data travels DOWN the stack — each layer adds a header (encapsulation). Across the internet, it travels as signals. At Host B, data travels UP — each layer removes its header (de-encapsulation).
Encapsulation: Each layer wraps the data from the layer above with its own header information. De-encapsulation: The reverse process — each layer at the receiver strips off its corresponding header.
Encapsulation — Adding headers at each layer when sending; De-encapsulation — Removing headers at each layer when receiving

Application Protocol

HTTP — HyperText Transfer Protocol

📖 What is HTTP?

HTTP (HyperText Transfer Protocol) is the protocol used to transfer web pages from a web server to a web browser.

  • Follows a client-server request/response model
  • The client (web browser) sends an HTTP request
  • The server processes it and sends back an HTTP response
  • Default port: 80
  • Data is sent in plaintext (no encryption)

🔄 Request/Response Model

Browser GET /index.html Server
Browser 200 OK + HTML Server

🔒 HTTPS — Secure Version

HTTPS (HyperText Transfer Protocol Secure) is HTTP with SSL/TLS encryption.

  • Data is encrypted before sending — anyone intercepting sees only ciphertext
  • Uses digital certificates to verify the server's identity
  • Default port: 443
  • Identified by 🔒 padlock in the browser address bar
  • Used for banking, e-commerce, logins, any sensitive data
HTTPS adds SSL/TLS encryption to protect data in transit
⚠ Exam warning: HTTP sends data in plaintext — vulnerable to man-in-the-middle attacks. HTTPS encrypts with SSL/TLS — always specify which one and WHY in exam answers.
HTTP — Protocol for transferring web pages using request/response model; Port 80 (HTTPS: Port 443)

Application Protocol

FTP — File Transfer Protocol

📁 What is FTP?

FTP (File Transfer Protocol) is used to upload and download files between a client and a server over a network.

  • Requires login credentials (username and password)
  • Can also allow anonymous access (e.g., public file downloads)
  • Default port: 21 (control connection)
  • Works well for large files and batch transfers

🔗 Two Separate Connections

FTP uses two separate channels for communication:

ConnectionPortPurpose
Control Connection21Sends commands (login, navigate, list files)
Data Connection20Transfers the actual file data
FTP uses separate control and data connections — commands and files travel on different channels
Exam tip: FTP is not secure — credentials and data are sent in plaintext. SFTP (SSH File Transfer Protocol) or FTPS (FTP over SSL) are encrypted alternatives.
FTP — File Transfer Protocol; uses separate control (port 21) and data (port 20) connections

Email Protocols: SMTP, POP3, IMAP

📤 SMTP

Simple Mail Transfer Protocol

  • Used to SEND email from a client to a mail server
  • Also used to relay email between mail servers
  • Default port: 25
  • Cannot retrieve email from a server (send only)
Client SMTP Server

📥 POP3

Post Office Protocol v3

  • Used to RETRIEVE email from a server
  • Downloads emails to ONE device
  • Typically deletes emails from server after download (can leave a copy)
  • Default port: 110
  • Best for single-device access
Server POP3 Client

🔄 IMAP

Internet Message Access Protocol

  • Used to RETRIEVE email from a server
  • Synchronises email across MULTIPLE devices
  • Keeps emails on the server (manages folders server-side)
  • Default port: 143
  • Best for multi-device access (phone, laptop, tablet)
Server IMAP Client

Common 4-mark question: "Compare POP3 and IMAP." Structure: (1) POP3 downloads to one device, IMAP syncs across multiple. (2) POP3 deletes from server, IMAP keeps on server. (3) POP3 gives full offline access, IMAP requires internet. (4) POP3 frees server space, IMAP uses server storage.

Application Protocol

BitTorrent Protocol

🧲 What is BitTorrent?

BitTorrent is a peer-to-peer (P2P) file-sharing protocol. Unlike traditional download (client-server), BitTorrent has no central server storing the file.

  • A file is split into many small pieces
  • Pieces are distributed across many peers
  • Users download pieces from each other simultaneously
  • Users also upload the pieces they already have to others

👥 Key Terms

Seeder
A peer that has the complete file and is uploading it to others
Leecher
A peer that is downloading (and usually uploading) pieces simultaneously

✅ Benefits

  • Reduces load on a single server — no single point of failure
  • Faster for popular files — more seeders = faster download
  • Scalable — more users means more upload bandwidth, not less
  • Resilient — if one peer goes offline, others still have the pieces
Peers directly exchange pieces of the file. The more peers, the faster the download!
⚠ Note: BitTorrent itself is legal technology. However, it can be used to share copyrighted material without permission, which is illegal in many jurisdictions.
BitTorrent — P2P file-sharing protocol; file split into pieces; no central server; seeder/leecher model

📝 Exam Tips — Protocols

Common Question 1

"Explain why a protocol is necessary for communication between two computers." [3]

Structure your answer (3 marks):

  • Definition: A protocol is an agreed set of rules governing data communication
  • Format agreement: Both devices must agree on how data is formatted/encoded so it is interpreted correctly
  • Timing/flow control: Prevents data loss when devices operate at different speeds
Common Question 2

"Describe the purpose of each layer in the TCP/IP protocol suite." [4]

Structure your answer (1 mark per layer):

  • Application: Provides network services to user applications; protocols like HTTP, FTP
  • Transport: Breaks data into segments; adds port numbers; provides reliable delivery (TCP) or fast delivery (UDP)
  • Internet: Adds IP addresses; routes packets across networks using routers
  • Link: Adds MAC addresses; controls transmission over physical medium (Ethernet, Wi-Fi)
Common Question 3

"Explain how encapsulation works when sending data over the internet." [4]

Structure your answer:

  • Data moves down the stack at the sender
  • Each layer adds its own header (encapsulation)
  • At receiver, data moves up the stack
  • Each layer removes its corresponding header (de-encapsulation)
Common Question 4

"Distinguish between POP3 and IMAP." [4]

Key differences:

  • POP3 downloads and deletes from server (single device)
  • IMAP keeps on server and syncs across multiple devices
  • POP3 better for offline access; IMAP needs internet
  • POP3 frees server storage; IMAP uses server storage
💡 Golden rule: For any "explain" question, always include: definition + how it works + why it matters. For "compare" questions, always mention both sides explicitly.

Section 2 — Switching Methods

Two Ways to Send Data Across a Network

🔗 Circuit Switching

A dedicated physical path (circuit) is established between sender and receiver for the entire duration of the communication.

  • Classic example: Traditional telephone calls
  • Path is reserved — no other traffic can use it
  • Data travels along this fixed path
A === dedicated line === B

📦 Packet Switching

Data is broken into small packets, each sent independently across the network (possibly via different routes), and reassembled at the destination.

  • Used by the Internet
  • No dedicated path — packets share network resources
  • Each packet includes a destination address
A P1→⦿ P2→⦿ P3→⦿ B
Circuit Switching — Dedicated path for entire communication. Packet Switching — Data split into packets, sent independently

Switching Method

Circuit Switching Explained

🔧 How It Works

Circuit switching establishes a dedicated physical communication path between sender and receiver for the entire duration of the call/transmission.

  1. Setup phase: A connection request is sent across the network
  2. Switches along the path reserve resources (bandwidth)
  3. A dedicated circuit is established end-to-end
  4. Data transfer: Data flows continuously along the fixed path
  5. Teardown: Circuit is released when communication ends

📞 Classic Example: Traditional Telephone Call

When you dial a phone number:

  • Phone network finds a path through exchanges
  • A physical circuit is created between you and the other person
  • The line is reserved — no one else can use it while you talk
  • When you hang up, the circuit is released
A fixed, dedicated circuit is established through network switches before data transmission begins
Circuit Switching — Establishes a dedicated physical path for the entire communication session

Circuit Switching — Benefits & Drawbacks

✅ Benefits

BenefitExplanation
Guaranteed bandwidthThe dedicated circuit ensures a fixed amount of bandwidth is available — no competition
Low latencyOnce the circuit is set up, there is minimal delay — data flows continuously
Data arrives in orderAll data follows the same path — no need for sequence numbers or reassembly
Good for real-time applicationsIdeal for voice calls and live video where consistent timing is essential

❌ Drawbacks

DrawbackExplanation
Inefficient — bandwidth wastedIf no data is being sent, the reserved line sits idle but no one else can use it
Setup timeTime is needed to establish the circuit before any data can be sent
Single point of failureIf any link in the circuit fails, the entire call is dropped
Not scalableRequires dedicated resources per connection — doesn't scale well for internet-sized traffic

📍 Where Circuit Switching Is Used

Traditional telephone networks (PSTN) — before VoIP, all phone calls used circuit switching. Modern phone networks have largely moved to packet switching (VoIP).

Exam Tip

When comparing circuit and packet switching, remember: circuit switching = dedicated resource → guaranteed quality but inefficient. Packet switching = shared resource → efficient but variable quality.

Switching Method

Packet Switching Explained

📦 How It Works

Packet switching breaks data into small packets and sends each one independently across the network.

  1. Original data/message is divided into packets
  2. Each packet contains a header + payload
  3. Packets are sent independently — possibly via different routes
  4. At destination, packets are reassembled into the original message

📝 Packet Structure

Each packet may take a different route through the network depending on congestion!
Packet Switching — Data split into packets, each sent independently with its own routing info

Packet Switching — Benefits & Drawbacks

✅ Benefits

BenefitExplanation
Efficient bandwidth useNo dedicated line — multiple users share the same network links; bandwidth is used only when data is actually being sent
Resilient / fault-tolerantIf one router or link fails, packets can be rerouted around the problem automatically
ScalableEasily scales to large networks like the Internet — adding more users doesn't require new dedicated lines
Multiple users share linesMany users can transmit simultaneously over the same physical infrastructure

❌ Drawbacks

DrawbackExplanation
Packets may arrive out of orderDifferent routes mean packets can arrive in the wrong order — need reassembly using sequence numbers
Variable delay (jitter)Network congestion causes unpredictable delays — not ideal for real-time applications like live voice/video without buffering
Overhead from headersEach packet carries header data (source/dest IP, seq #, etc.) — this is overhead that doesn't carry user data
Packet loss requires retransmissionIf packets are lost (congestion, errors), they must be retransmitted — adds delay

📍 Where Packet Switching Is Used

The Internet is entirely packet-switched. It is used for: web browsing (HTTP), email (SMTP/POP3/IMAP), file transfer (FTP), streaming (Netflix, YouTube), VoIP (Skype, Zoom), and online gaming.

Networking Hardware

The Role of the Router

📌 What Does a Router Do?

A router is a networking device that forwards data packets between computer networks. Its key functions:

  • Examines the destination IP address in each packet's header
  • Determines the best next path/hop to forward the packet
  • Connects different networks together (e.g., your home LAN to the Internet)

🗺️ Routing Tables

Routers use routing tables to make forwarding decisions. A routing table contains:

Destination NetworkNext HopMetric (Cost)
192.168.1.0/24Directly connected0
10.0.0.0/1610.0.0.11
0.0.0.0/0203.0.113.110

The router chooses the best route (lowest metric/cost) for each packet.

🔄 Different Routes for Different Packets

Packets from the same message can be routed differently depending on network conditions:

  • If Route A is congested, the next packet may take Route B
  • This is why packets may arrive out of order
  • This dynamic routing makes the network resilient and efficient
Routers use routing tables to determine the best next hop for each packet
Remember: A switch forwards data within a single network (using MAC addresses). A router forwards data between different networks (using IP addresses).
Router — Device that forwards packets between networks using IP addresses and routing tables

Packet Switching in Action Across the Internet

The original message is split into packets, each travels independently, may arrive out of order, and is reassembled using sequence numbers in the header

Revision Summary

📌 Topic Summary — Key Takeaways

📋 Protocols & TCP/IP Stack

  • A protocol is a set of rules governing data communication
  • Protocols are organised into layers — separation of concerns
  • The TCP/IP suite has 4 layers: Application, Transport, Internet, Link
  • Each layer encapsulates data by adding headers when sending; de-encapsulates when receiving
  • Application: provides network services to apps (HTTP, FTP, SMTP, etc.)
  • Transport: segments data + port numbers + TCP (reliable) vs UDP (fast)
  • Internet: IP addresses + routing packets across networks
  • Link: MAC addresses + physical transmission (Ethernet, Wi-Fi)

📨 Common Protocols

  • HTTP (port 80) — transfers web pages; HTTPS (port 443) — secure version
  • FTP (port 21) — file transfer; separate control & data connections
  • SMTP (port 25) — send email; POP3 (110) — download to one device; IMAP (143) — sync across devices
  • BitTorrent — P2P file sharing; file split into pieces; seeder/leecher model

🔗 Circuit Switching

  • Dedicated physical path for entire communication
  • ✅ Guaranteed bandwidth, low latency, in-order delivery
  • ❌ Inefficient (idle line wasted), setup time, not scalable
  • Used in traditional telephone networks

📦 Packet Switching

  • Data split into packets, each sent independently
  • ✅ Efficient bandwidth, resilient (reroutes around failure), scalable
  • ❌ Out-of-order arrival, variable delay (jitter), header overhead
  • Used by the Internet

🔄 Router's Role

  • Examines destination IP address in each packet
  • Uses routing tables to find the best next hop
  • Forwards packets between different networks
  • Different packets from the same message may take different routes
💡 Exam-ready checklist: Can you define a protocol? List the 4 TCP/IP layers with their functions? Explain encapsulation? Compare circuit vs packet switching? Name common protocols and their ports? Describe a router's role?
